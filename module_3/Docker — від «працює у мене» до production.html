// 02 · дані

Дані всередині контейнера

Контейнер ефемерний. Все, що записано у writable layer — зникає при rm чи redeploy. Використовуй volumes.

/app/data → 🗑

docker exec для фіксу

Поправив файл всередині запущеного контейнера? Це не фікс — це міна. Зміни йдуть у Dockerfile або код.

exec → inspect only
+ забуте --rm + .dockerignore + COPY . перед deps + публікація db-портів + запуск як root

// 02 layer caching

Стабільне → мінливе

Залежності копіюй до коду. Тоді pip install кешується, поки requirements.txt не змінюється.

# multi-stage · production pattern FROM python:3.11-slim AS builder WORKDIR /build COPY requirements.txt . RUN pip install --prefix=/install -r requirements.txt FROM python:3.11-slim AS production WORKDIR /app COPY --from=builder /install /usr/local COPY src/ . USER app CMD ["python", "main.py"] # builder має компілятори # production — тільки код + deps # менша атака · менший образ
// slim base + ordered layers + multi-stage = production-grade
